# Supabase setup

1. Open the Supabase SQL Editor and run `supabase/schema.sql`.
2. Run `supabase/commerce-upgrade.sql` to add synced carts, wishlists, addresses, payments, tracking events, verified product reviews, and recurring orders.
3. In Authentication settings, enable Email authentication.
4. Set the Site URL to the deployed application URL and add local/Vercel callback URLs, including `http://localhost:3000/auth/callback`.
4. Phone Auth is optional and is not required by this application. The signup phone field is profile information only.
5. The schema creates a public Storage bucket named `simba-assets`.
6. Upload the landing video to `simba-assets/Landing/`.
7. Open the uploaded object, choose **Get URL**, and copy its public URL.
8. Set that URL as `NEXT_PUBLIC_HERO_VIDEO_URL`.

## Google Auth

Google sign-in is available for customer accounts through Supabase OAuth. Staff accounts continue to use Email Auth so the selected staff role can be checked before dashboard access.

1. In Google Cloud Console, create OAuth web credentials.
2. In Supabase, open **Authentication > Providers > Google**.
3. Add the Google OAuth client ID and its matching client secret, then enable the provider.
4. Add this Supabase callback URL to Google Cloud's **Authorized redirect URIs**:

   `https://hmpygqkpgmcyhsamhosp.supabase.co/auth/v1/callback`

5. Keep the Google client secret only in the Supabase dashboard. It is not a frontend or Vercel environment variable.

A value ending in `.apps.googleusercontent.com` is a Google client ID. The matching client secret is a different value generated by Google Cloud.

## Staff accounts

Staff accounts are separate from customer accounts. Do not use a customer's Auth user for staff access.

Create a new staff user in Supabase Authentication first. Then assign that separate user's role in the SQL Editor:

```sql
update public.profiles
set role = 'manager', branch_id = 'kigali-kic'
where email = 'manager@example.com';
```

Valid staff roles are `manager`, `admin`, `driver`, and `ceo`. The staff checkbox only requests a staff workspace; access is granted only when the database profile has the matching role.

## Optional Phone Auth

The application uses Email Auth for customers, managers, administrators, drivers, checkout, and protected dashboards. Supabase Phone Auth is not required.

Phone/SMS authentication may require a paid Supabase plan and a configured SMS provider. It can be enabled later without changing the current email authentication flow.

Use a public object URL for a permanent landing video. Signed URLs are suitable for private files but can expire.

Keep `SUPABASE_SERVICE_ROLE_KEY` server-only. Never prefix it with `NEXT_PUBLIC_` or expose it in browser code.
